# 路线图

HALF 目前处于 `v0.x` 阶段。项目可用，但接口、数据模型细节和部分运行行为在次版本间仍可能变化。本路线图旨在传达方向和优先级，而非保证确切日期或范围。

[English](./ROADMAP.md) | [简体中文](./ROADMAP.zh-CN.md)

## 当前重点

当前重点是让外部用户更容易理解、试用和贡献 HALF：

- 保持核心 human-in-the-loop 多 agent 工作流的可靠性
- 改善首次运行的清晰度和上手体验
- 标准化可复用的任务、工作流和 handoff 约定
- 完善公开文档、设计记录和发布纪律

## v0.3

计划重点：

- 将任务输入输出标准化为结构化任务卡片
- 用可复用工作流预设演进现有的流程模板模型
- 定义直连任务节点间的轻量化 handoff 格式
- 增加移动端友好的任务状态通知支持
- 改进管理员管理的公共 agent 目录的共享行为
- 通过文档和演示素材改善首次试用的可操作性

详细的 v0.3 执行范围、非目标、依赖和验收说明见 `docs/roadmap/v0.3.md`（即将推出）。纯研究工作记录在 `docs/research/`（即将推出），持久架构决策记录在 `docs/adr/`（即将推出）。

v0.3 不预期成为：

- 稳定的多租户 SaaS 版本
- 全自动 agent runner
- 完整上下文组装引擎
- 支持分支、循环或动态路由的工作流引擎
- 每个 API 形状和内部模型的长期兼容性保证

## v0.4+ 方向性主题

以下是方向性主题，而非固定承诺。其范围和优先级将根据 v0.3 实现结果、研究反馈、企业试点发现和开源维护能力进行调整。

- **实验与评估。** 增加实验运行日志、失败或经验卡片，以及工作流执行与评估信号之间的更好关联。
- **技能驱动工作流。** 引入技能注册表，将可复用技能绑定到工作流模板。
- **外部化状态。** 在 v0.3 任务卡片和 handoff 格式经实际使用验证后，探索累积外部化状态。
- **治理钩子。** 探索受控 AI 编码工作流的策略检查和人工审查点。
- **试点驱动的企业记录。** 使用企业试点报告来决定可复用企业案例记录应成为文档、实验记录还是产品功能。
- **合规 agent 集成研究。** 探索未来版本是否以及如何通过显式、用户配置的、官方支持的基于 API 的集成来支持自动化 agent 调用。HALF 不会依赖私有 API、UI 自动化、逆向工程或凭证共享来自动化订阅制 coding agent。

需求到测试的工作流被视为重要的工作流模板示例，而非独立的产品线。

## 稳定性边界

在 `v0.x` 期间：

- API 字段仍可能演进
- 数据库和运行行为仍可能调整
- 文档和工作流约定可能继续完善

评估 HALF 的用户应将当前版本视为早期开源版本，用于受控的自托管使用、反馈和迭代。

## 暂时不在范围内

以下不是当前路线图的优先事项：

- 将 HALF 定位为通用项目管理替代品
- 通过私有 API、UI 自动化、逆向工程或凭证共享来自动化订阅制 coding agent
- 承诺加固的多租户隔离
- 添加不直接改善可用性、可靠性或贡献者上手的路线图项目
