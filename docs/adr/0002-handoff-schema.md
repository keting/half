# ADR 0002: 轻量化 Handoff Schema

状态: proposed

## 背景

HALF 跨多个 AI coding agent 和人工操作员协调工作。任务结果应能将结构化上下文传递给直接依赖的任务，而无需操作员手动检查每个制品。

## 决策

v0.3 将使用轻量化 handoff schema 和直接前驱 handoff 在后继提示中的包含。

这是结构化前驱任务输出在后继提示中的合法包含，而非提示注入攻击。

初始字段：

- `summary`
- `modified_files`
- `artifact_references`
- `test_status`
- `known_risks`

HALF 应在任务结果目录中读写 `handoff.json`，在任务详情中只读展示，并在下游提示中包含直接前驱任务的定字段。

## 字段约束

具体字段类型、最大长度和截断限制将在实现期间定稿。后继提示 handoff 包含可能截断长文本，但必须保留制品引用或提供指向完整 handoff 制品的清晰指针。

## 非目标

v0.3 不提供：

- 用户可编辑的包含预览
- 工作流特定的包含策略
- 自动摘要、合并、去重或冲突检测
- 累积外部化状态集成
- 通用上下文组装引擎

## 备选方案

- 仅在 HALF 中展示 handoff 数据而不包含在后继提示中。
- 在 v0.3 中构建完整上下文组装引擎。
- 在无结构化 handoff schema 的情况下将所有前驱制品包含在后继提示中。

## 后果

这为 v0.3 带来实际执行收益，同时保持实现有界。后续版本可能基于观察到的 handoff 数据引入工作流特定的上下文组装。
