# ADR 0001: 任务卡片 Schema

状态: proposed

## 背景

HALF 需要更结构化的任务单元来支持可复用工作流模板、handoff、实验记录和未来评估工作流。现有任务生命周期已提供任务状态、分配、依赖和结果检测。v0.3 应扩展现有模型，而非引入并行任务系统。

## 决策

为 v0.3 定义与现有任务模型和任务生命周期兼容的最小任务卡片 schema。

Schema 应优先支持以下字段：

- 流程模板预设
- 提示生成
- 轻量化 handoff
- 未来实验运行日志

## Schema 设计起点

本 ADR 不定稿 v0.3 任务卡片字段列表。实现设计应从字段类别而非固定 schema 开始。

可能的类别包括：

- 任务标识和可读描述
- 意图或目标
- 所需输入和引用
- 预期输出或完成信号
- 验收标准或验证提示
- 约束、假设和风险说明
- 分配和角色信息
- 下游任务的 handoff 预期

字段名称、必需或可选状态、存储位置和迁移策略必须在本 ADR 提升为 accepted 前决定。

## 约束

- 不要创建第二个任务系统。
- 保留现有任务生命周期行为。
- 保持 schema 足够小，以便在 v0.x 期间安全迁移。
- 避免仅为推测性未来自动化服务的字段。

## 备选方案

- 保持任务主要为自然语言描述，推迟标准化。
- 引入与现有任务模型并行的独立任务卡片模型。
- 一步设计完整的实验导向任务 schema。

## 后果

任务卡片 schema 成为 v0.3 工作流和 handoff 工作的共享基础。对此 schema 的更改应作为产品和架构决策审查，而非仅作为 UI 变更。
