# HALF 新手参与路径

[English](./newcomer-path.md) | [简体中文](./newcomer-path.zh-CN.md)

这份文档面向刚接触 GitHub、开源协作或 HALF 的贡献者。你不需要一开始就写
代码；复现问题、补充文档、整理截图和提出研究型讨论也都是有效贡献。

## 第一档：15 分钟了解项目

- 阅读 [`README.zh-CN.md`](../README.zh-CN.md)。
- 浏览 README 中的产品截图和最小闭环 GIF。
- 浏览 [`ROADMAP.zh-CN.md`](../ROADMAP.zh-CN.md)，了解当前方向。
- 浏览 Issues 和 Discussions，观察已有问题和讨论方式。

完成这一档后，你应该能回答：

- HALF 是什么，不是什么。
- HALF 为什么采用 human-in-the-loop，而不是自动执行 agent。
- 当前 roadmap 大致关注哪些方向。

## 第二档：半天跑通项目

- 按 [`docs/quickstart.zh-CN.md`](./quickstart.zh-CN.md) 启动本地环境。
- 登录 Demo Project。
- 浏览 Plan DAG、任务看板、任务详情和 handoff prompt。
- 阅读 [`docs/project-structure.md`](./project-structure.md)，了解主要代码目录。
- 记录你遇到的疑问、文档缺口、不顺手的体验或错误提示。

这些记录都可以变成贡献：文档改进、bug report、UI/UX feedback、FAQ 或
Discussion。

## 第三档：第一个贡献

适合作为第一次贡献的事项：

- 修复 typo 或不清楚的文档表述。
- 补充 Quick Start 中遗漏的步骤、截图或故障排除信息。
- 复现一个 open bug，补充复现步骤、环境信息、日志或截图。
- 验证一个 PR 是否真的解决了对应 Issue。
- 选择 `good first issue` 标签下的小任务。

提交第一个 PR 前，请确认：

- 改动范围足够小。
- PR 描述说明了为什么改、改了什么、如何验证。
- 文档、截图或测试更新与本次改动直接相关。
- 没有包含密钥、私有 URL、access token 或个人本机路径。

## 第一个 PR 之后

完成第一个 PR 后，可以继续尝试：

- 认领 `status:ready` 的工程任务。
- 在 Discussion 提出研究型想法或 roadmap 建议。
- 贡献 workflow 模板、handoff prompt 或 plan DAG case。
- 记录 agent 协作失败模式、人工介入点或 benchmark / evaluation case。
- 参与 Issue triage、PR Review 和 Milestone 讨论。

如果你不确定一个想法应该进入 Issue、Discussion 还是 PR，请先阅读
[`CONTRIBUTING.zh-CN.md`](../CONTRIBUTING.zh-CN.md) 中的判断规则。
